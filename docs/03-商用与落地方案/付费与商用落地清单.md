## 二、付费与计费（待实施）

### 2.1 推荐模型：预充值 + 按次/按量扣费

- 用户先充值余额或「点数」，使用 AI 生成时按次或按 token 扣减。
- 本地生成可继续免费；导出报告可免费或计入套餐。

### 2.2 后端需具备的能力（与收费相关）

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 用户与登录 | 注册/登录/找回密码，登录后颁发 Token（如 JWT），前端请求 AI/支付时携带 | P0 |
| 账户与计费 | 用户表、余额/点数表、流水表；扣费时事务内「扣余额 + 记流水」；支付回调幂等 | P0 |
| 套餐与定价 | 可先固定「每次 AI 生成 = N 元或 M 点」，后续再做包月、不同模型不同价 | P1 |
| 支付渠道 | 国内：微信/支付宝（需企业主体）；海外：Stripe；可先接一种 | P1 |

### 2.3 前端待配合项（付费上线时）

- 若后端提供「当前用户余额/点数」接口，前端可增加：
  - 顶部或侧栏展示余额；
  - AI 生成前校验余额，不足时提示充值；
  - 「充值」入口跳转支付页或打开支付弹窗。
- 移除或隐藏「用户自行填写 API Key」的 UI，统一走后端代理与计费（可选：保留「自带 Key 加密托管」入口，由后端提供绑定接口）。

### 2.4 定价思路（示例）

- 本地生成：免费。
- AI 生成：按次（如 0.1 元/次）或按 token；新用户送若干次免费额度。
- 导出报告：免费或计入套餐。

## 四、环境变量与安全（生产必查）

- **前端**  
  - `VITE_APP_SKIP_LOGIN`：生产应为 `false` 或未设置。  
  - `VITE_APP_DEV_BACKDOOR`：生产勿设为 `true`。  
  - `VITE_APP_API_BASE`：指向正式后端（如 https://api.plc-sim.com）。
- **后端**  
  - API Key 仅存于环境变量或密钥管理服务，不写入前端包、不通过接口返回。  
  - 生产关闭或限制 source map 访问。

## 五、小结

- **当前**：合规与稳定性已做一轮优化（后门关闭、免责声明、协议与隐私入口、错误边界、版本与构建时间）；后端已具备 AI 代理，可为付费打基础。
- **付费落地**：需后端先具备用户体系、账户与流水、支付回调；前端再增加余额展示、充值入口与（可选）移除自带 Key 输入。按本清单与《商用落地方案》分阶段推进即可。
