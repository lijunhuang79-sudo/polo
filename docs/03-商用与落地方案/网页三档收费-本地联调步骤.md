# 网页三档收费 — 本地联调步骤

本文档用于在本地验证「已实现的前端三档逻辑」：免费档、基础版激活、AI 周卡链接、权限控制。**未接后端时，激活与 AI token 会走本地模拟（任意授权码通过、任意 t= 给 7 天）。**

---

## 一、环境准备

### 1.1 启用三档开关

在项目根目录的 **`.env`** 中增加（或确认已有）：

```bash
# 三档收费：本地联调时设为 true
VITE_APP_TIERED_PAYWALL=true
```

若不想改 `.env`，可新建 **`.env.local`**（优先级更高），内容仅需：

```bash
VITE_APP_TIERED_PAYWALL=true
```

### 1.2 启动开发服务

```bash
npm run dev
```

浏览器打开终端提示的地址（一般为 `http://localhost:5173`）。  
若已配置 `VITE_APP_SKIP_LOGIN=true`，会直接进入主界面，无需输入调试密钥。

---

## 二、联调用例（按顺序执行）

### 用例 1：免费档 — 仅 2 个场景可点

**前置**：清空本地与该站相关的存储，或使用无痕窗口，确保未激活基础版、无 AI token。

**步骤：**

1. 打开应用，进入「场景需求描述」区域。
2. 点击 **「典型场景示例」** 展开列表。
3. **预期**：
   - **可点击**：仅 **「基础: 启保停控制」**、**「基础: 延时启动」** 两项；点击后文案填入输入框。
   - **不可点击**：其余场景（双次启动、星三角、正反转、红绿灯等）为置灰 + 锁图标，hover 提示「升级基础版 9.9 元解锁」。
4. **本地生成 / AI 智能生成**：免费档下若未激活基础版，应只显示「升级基础版」相关提示；「一键生成 PLC 程序」为禁用状态，文案为「升级后可用」。

**通过标准**：免费用户只能使用上述 2 个场景示例，这两个场景可以用本地生成完整功能。其他场景无法使用本地生成与 AI 生成。

---

### 用例 2：基础版激活（占位：任意授权码通过）

**步骤：**

1. 在「场景需求描述」区域找到 **「已有授权码？激活」**，点击展开输入框。
2. 输入任意非空字符串（如 `test-license-001`），点击 **「激活」**。
3. **预期**：
   - 无后端时：请求可能失败，但当前代码 **catch 后会在本地模拟通过**，输入框收起，页面表现为「已激活基础版」。
   - 有后端且授权码有效时：返回成功，同样收起输入框并写入本地存储。

4. **验证基础版生效**：
   - 再次展开「典型场景示例」：**全部场景**应可点击，无锁。
   - 切换为 **「本地生成」**，输入或选择任意场景文案，点击 **「一键生成 PLC 程序」**：应能正常生成并进入 I/O、程序、仿真等区域。
   - 若未购买 AI 周卡，「AI 智能生成」旁应显示「AI 周卡 19.9 元」或类似文案，点击可切换但生成按钮仍为「购买 AI 周卡后可用」或禁用。

**通过标准**：激活后全部场景可点、本地生成可用；AI 生成仍受「是否有有效 AI token」控制。

---

### 用例 3：AI 周卡链接（占位：带 `?t=xxx` 给 7 天）

**步骤：**

1. 新开一个标签页或无痕窗口，访问：  
   `http://localhost:5173/?t=demo-ai-token-123`  
   （端口以实际为准。）

2. **预期**：
   - 页面加载后会请求 `/api/ai-token/validate?t=demo-ai-token-123`。
   - 无后端时：请求失败，代码 **catch 后本地模拟**，写入 AI token 与 7 天有效时间；页面表现为「AI 周卡有效」。
   - 有后端且 token 有效时：返回 `{ valid: true, validUntil: "..." }`，同样写入本地。

3. **验证 AI 周卡生效**：
   - 若当前已激活基础版：应能切换到 **「AI 智能生成」**，且「一键生成 PLC 程序」可用（不再显示「购买 AI 周卡后可用」）。
   - 若当前未激活基础版：根据方案，AI 周卡依赖基础版，前端上「AI 智能生成」可能仍提示需先开通基础版（以你当前 UI 逻辑为准）。

**通过标准**：带 `?t=任意值` 打开后，在「已激活基础版」的前提下，AI 生成可用。

---

### 用例 4：三档展示与按钮状态汇总

| 状态           | 典型场景示例     | 本地生成 | AI 智能生成     | 一键生成按钮   |
|----------------|------------------|----------|------------------|----------------|
| 免费档         | 仅 2 个可点      | 不开放   | 不开放           | 禁用，升级后可用 |
| 已激活基础版   | 全部可点         | 开放     | 显示「购买 19.9」| 本地可用，AI 需周卡 |
| 基础版 + AI 有效 | 全部可点       | 开放     | 开放             | 均可使用       |

在本地依次切换「未激活 / 激活基础版 / 带 t= 打开」三种情况，核对表格中的展示与按钮状态。

---

### 用例 5：清除本地状态后重测免费档

**步骤：**

1. 打开开发者工具 → Application（应用）→ Local Storage → 选中当前站点。
2. 删除以下 key（若存在）：
   - `plc_tier_basic_license`
   - `plc_tier_ai_token`
   - `plc_tier_ai_valid_until`
3. 刷新页面。

**预期**：恢复为免费档；仅 2 个场景可点，本地生成与 AI 生成不可用，与用例 1 一致。

## 三、可选：对接真实后端时

当后端提供以下接口时：

- `POST /api/license/activate`（body: `{ licenseKey: "xxx" }`）
- `GET /api/license/validate`（query: licenseKey 或 header X-License-Key）
- `GET /api/ai-token/validate?t=xxx`
- `POST /api/order/create`（body: `{ productType: "basic" | "ai_week", basicLicenseKey?: "xxx" }`）
- `GET /api/order/status?orderId=xxx`
- `POST /api/order/simulate-callback`（body: `{ orderId: "xxx" }`，联调用：模拟支付成功并发放授权码/token）

在 **`.env`** 或 **`.env.local`** 中配置：

```bash
VITE_APP_API_BASE=http://localhost:3001
```

将 `3001` 换成实际后端端口（本项目 backend 默认 3001）。前端会请求上述接口。  
联调时用无效授权码 / 无效 token 测一次，确认前端能正确展示「授权码无效」「链接已失效」等错误提示。

### 使用本项目 backend 联调（SQLite 持久化）

1. 在 **backend** 目录启动后端：`cd backend && npm run dev`（或 `node server.js`），默认监听 **3001** 端口。
2. 前端项目根目录 `.env` 中设置 `VITE_APP_API_BASE=http://localhost:3001`，并确认 `VITE_APP_TIERED_PAYWALL=true`，然后 `npm run dev`（前端会通过 Vite 代理把 `/api` 转到 3001）。
3. **支付后获取激活码（必测）**：
   - 打开 http://localhost:5173，进入购买流程：点击「升级基础版」或「购买 AI 周卡」→ 弹窗中点「去支付」→ 出现订单号与收款码。
   - **若已用支付宝完成付款**：因当前未接入支付回调，服务器不知道已付款。请点击 **「模拟到账（联调）」**，页面会自动查询并弹出「购买成功」，显示**激活码**（基础版）或 **AI 周卡链接**（AI 周卡），可复制保存。
   - 若未付款仅联调：直接点击「模拟到账（联调）」即可补发授权码并弹出激活码。
4. **基础版联调**：也可用接口：`POST /api/order/create` body `{"productType":"basic"}` 得 `orderId`；`POST /api/order/simulate-callback` body `{"orderId":"..."}` 得 `license_key`。在「已有授权码？激活」处输入该 `license_key`，应激活成功。
5. **AI 周卡联调**：`POST /api/order/create` body `{"productType":"ai_week"}` 得 `orderId`；`POST /api/order/simulate-callback` body `{"orderId":"..."}` 得 `token` 和 `validUntil`。浏览器打开 `http://localhost:5173/?t=返回的token`，应表现为 AI 周卡有效。

后端使用 **SQLite** 持久化（`plc-sim.db`），重启后订单与授权数据保留。

## 四、常见问题

- **看不到「已有授权码？激活」或典型场景锁**  
  检查 `VITE_APP_TIERED_PAYWALL=true` 是否生效（改 .env 后需重启 `npm run dev`）。

- **激活后刷新又变回免费档**  
  检查 Local Storage 中是否有 `plc_tier_basic_license`；若没有，说明激活未写入或存储被清空。

- **带 `?t=xxx` 打开后仍无 AI 权限**  
  先确认是否已激活基础版（方案约定：AI 周卡仅在基础版开通后可用）；再检查 `plc_tier_ai_valid_until` 是否写入且为未来时间戳。

- **本地付款界面/购买弹窗出不来**  
  ① **硬刷新**：按 `Ctrl+Shift+R`（Windows）或 `Cmd+Shift+R`（Mac），清缓存并重新加载。  
  ② **确认入口**：在「典型场景示例」下方应能看到「升级基础版 9.9 元」「购买 AI 周卡 19.9 元」等入口，点击后才会弹出购买弹窗；若已激活基础版且已有 AI 周卡，这些入口可能被收起，可用无痕窗口或清除 Local Storage 中 `plc_tier_basic_license`、`plc_tier_ai_token`、`plc_tier_ai_valid_until` 后再试。  
  ③ **控制台**：打开开发者工具（F12）→ Console，看是否有报错；有报错时按提示排查（如接口地址、后端未启动等）。  
  ④ 重启前端：终端里停掉 `npm run dev` 后重新执行一次。
