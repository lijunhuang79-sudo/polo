# 针对西门子 S7-200 SMART 的优化方案

**文档版本**：1.0  
**适用**：PLC 编程仿真器生成的程序用于 S7-200 SMART 实机  
**最后更新**：按实际修订日期填写  

---

## 1. 已对齐与建议保持的规范

| 项目 | 当前状态 | 说明 |
|------|----------|------|
| 指令集 | STL 使用 LD / A / O / AN / = / TON 等 | 与 S7-200 SMART 兼容，无需改 |
| 定时器 T37 | TON T37, 50 → 50×100ms = 5s | T37 为 100ms 时基，符合 S7-200 SMART 规范 |
| I/O 地址 | I0.x, Q0.x, M0.x | 与 S7-200 SMART 寻址一致 |
| 定时器编号 | 优先使用 T37–T63（100ms） | 避免与 1ms/10ms 区段混用，便于实机一致 |

---

## 2. 已实施的优化（代码侧）

### 2.1 BOM 默认 CPU 型号

- **原**：默认 `CPU 224XP (或同级 S7-1200)`。  
- **现**：默认推荐 **S7-200 SMART** 常见型号（如 ST20 / SR20），便于直接选型与实机一致。  
- **位置**：`src/services/plcLogic.ts` 中标准 BOM 第一项、`aiSolutionValidator.ts` 默认 hardware、AI 系统提示中的 model 示例。

效果：生成的方案清单中，PLC 主机型号更贴近 S7-200 SMART，减少现场选型歧义。

### 2.2 定时器使用约定（文档化）

- 本应用生成的 **TON** 定时器统一使用 **100ms 时基** 编号（S7-200 SMART 中 **T37–T63**）。  
- 示例：`TON T37, 50` 表示 5.0 秒；`TON T38, 30` 表示 3.0 秒。  
- 若场景中需要多个定时器，建议依次使用 T37、T38、T39…，避免与 T32（1ms）、T33–T36（10ms）混用，便于在 STEP 7-Micro/WIN SMART 中与实机行为一致。

---

## 3. 建议新增的优化（可选实现）

### 3.1 导出「仅 STL 文本」便于粘贴到 STEP 7-Micro/WIN SMART

- **需求**：用户将本应用生成的 STL 复制到 STEP 7-Micro/WIN SMART 的 STL 视图中手工录入时，希望得到**纯文本、无 HTML、无多余注释**的 STL。  
- **方案**：在方案展示区（STL 标签页）增加按钮「**复制 STL（纯文本）**」，将当前 `stlCode` 以纯文本形式写入剪贴板（可去掉或保留行尾 `// 注释`，由产品决定）。  
- **价值**：减少从网页复制时的格式问题，加快在 STEP 7-Micro/WIN SMART 中的录入速度。

### 3.2 可选「S7-200 SMART 型号」预设

- **需求**：不同 S7-200 SMART 型号 DI/DO 点数不同（如 SR20 12DI/8DO，CR40 24DI/16DO），用户希望生成的 I/O 表不超出实机点数。  
- **方案**：在设置或场景选择处增加可选「**目标 PLC 型号**」：如 SR20/ST20、CR40 等。生成 I/O 表时根据所选型号限制最大 DI/DO 数量，并在超出时给出提示（如「当前方案需要 14 点 DI，SR20 仅支持 12DI，请减少信号或选择 CR40」）。  
- **价值**：避免方案 I/O 超出实机容量，减少现场改线或换型。

### 3.3 交通灯等长周期定时器的时间基准说明

- **现状**：交通灯等场景使用 `TON T37, 120`（12.0 秒）等，T37 为 100ms 时基，最大值 32767×0.1s ≈ 3276.7s，满足需求。  
- **优化**：在导出报告或 STL 注释中增加一行说明，例如：`// T37 为 100ms 时基，设定值 120 = 12.0s，适用于 S7-200 SMART`。  
- **价值**：用户在手输或核对时更清楚与实机定时器行为一致。

### 3.4 计数器编号约定

- **建议**：生成的 **CTU/CTD** 使用 S7-200 SMART 支持的计数器编号 **C0–C255**。当前若使用 C1、C2 等，保持即可；若将来扩展更多计数器，避免使用超出 C255 的编号。  
- **文档**：在《APP程序导入西门子S7-200-SMART说明》或本方案中注明「定时器建议 T37–T63，计数器 C0–C255」。

## 4. S7-200 SMART 资源速查（供生成与校对用）

| 资源 | 范围/说明 |
|------|------------|
| 定时器 100ms 时基 | T37–T63（本应用 TON 建议用此段） |
| 定时器 10ms 时基 | T33–T36 等 |
| 定时器 1ms 时基 | T32, T96 等 |
| 计数器 | C0–C255 |
| 典型型号 DI/DO | SR20/ST20：12DI / 8DO；CR40：24DI / 16DO；更多见产品手册 |

---

## 5. 小结

- **已做**：BOM 默认推荐 S7-200 SMART 型号；定时器使用 T37 等 100ms 区段并在文档中约定。  
- **建议做**：增加「复制 STL（纯文本）」、可选「目标 PLC 型号」限制 I/O 点数、在导出/注释中标明 T37 时基说明、计数器 C0–C255 约定。  
- **不改**：现有 STL/LAD/SCL 指令集与地址格式已兼容 S7-200 SMART，无需为兼容性大改逻辑。

按上述优化后，本应用生成的程序在**手工录入到 STEP 7-Micro/WIN SMART 并下载到 S7-200 SMART** 时，会更贴近实机资源与习惯，减少现场调整。
