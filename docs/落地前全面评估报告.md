## 三、安全与合规评估

### 3.1 高风险项 🔴

| 风险 | 现状 | 建议 |
|------|------|------|
| **API Key 暴露** | 用户在前端输入 Key，存 localStorage，浏览器直连 DeepSeek/Gemini 等 | **P0**：前端改为调用自有后端 `/api/ai/generate`，后端持 Key，前端永不接触 |
| **开发者后门** | 生产环境已关闭 `password === 'a'`，但 `DEV_DEBUG_PASSWORD` 仍存在 | 确保生产构建不包含后门逻辑；可考虑完全移除 |
| **CORS 配置** | 后端 `Access-Control-Allow-Origin: *` | 商用应限制为 `https://www.plc-sim.com` 等白名单 |
| **无请求鉴权** | `/api/ai/generate` 无 Token 校验 | 任何人可调接口，无法计费与限流 |

### 3.2 中风险项 🟡

| 风险 | 现状 | 建议 |
|------|------|------|
| 用户协议/隐私政策 | 占位内容，未法务审阅 | 正式发布前由法务定稿 |
| 软著与备案 | 未体现 | 中国大陆运营需 ICP 备案；建议申请软著 |
| 数据留存 | 无明确策略 | 在隐私政策中写明保留期限与删除机制 |

### 3.3 已缓解项 ✅

| 项目 | 状态 |
|------|------|
| 生产环境后门 | 已禁用（需显式 VITE_APP_DEV_BACKDOOR=true） |
| 工业/教学免责 | 页脚已声明 |
| 错误边界 | 已实现，避免白屏 |
| HTTPS | 生产环境已配置 |

## 四、技术架构评估

### 4.1 架构现状

```
┌─────────────────┐         ┌──────────────────┐         ┌─────────────────┐
│   浏览器 SPA    │ ──────> │  DeepSeek/Gemini  │         │  自有后端       │
│  (用户输入 Key) │  直连   │  OpenAI 等 API    │         │  (Key 在 env)   │
└─────────────────┘         └──────────────────┘         └────────┬────────┘
        │                                                          │
        │ 未接入                                                    │ /api/ai/generate
        └─────────────────────────────────────────────────────────┘
```

- **前端**：可选择「本地生成」或「AI 生成」；AI 模式下用户输入 Key，前端直连第三方 API。
- **后端**：已有 `/api/ai/generate`、`/api/ai/test`，从 env 读取 Key，但**前端未使用**。

### 4.2 技术债

| 项目 | 说明 |
|------|------|
| 前后端 AI 路径不统一 | 前端 aiGenerator 直连；后端 server.js 有完整实现，需统一为「前端只调后端」 |
| 配置硬编码 | 模型列表、API 地址等写死在前端，调整需重新构建 |
| 无接口文档 | 后端 API 无 OpenAPI/Swagger，协作成本高 |
| 无自动化测试 | 无单测、E2E，回归依赖人工 |

## 五、用户体验评估

### 5.1 优点 ✅

- 界面清晰：场景 → 生成 → I/O → 代码 → 仿真，流程顺畅
- 本地生成零门槛：无需 Key 即可体验
- AI 多模型可选：满足不同用户偏好
- 仿真与 HMI 直观：电机、灯、按钮等可视化
- 错误有兜底：AI 失败时回退到本地生成

### 5.2 待改进 ⚠️

| 项目 | 建议 |
|------|------|
| 首次使用引导 | 增加简短「新手指引」或示例场景一键填充 |
| 加载与错误提示 | AI 生成时增加进度/预估时间；错误信息可更友好 |
| 移动端适配 | 当前偏桌面端，移动端可优化 |
| 帮助与反馈 | 增加 FAQ、使用说明、反馈入口 |

## 七、落地建议与优先级

### 7.1 分阶段路线图

| 阶段 | 目标 | 关键动作 | 预估 |
|------|------|----------|------|
| **Phase 0** | 内测/演示可用 | 保持现状，仅限内部或邀请用户 | 当前 |
| **Phase 1** | 安全合规可对外 | 前端改调后端 AI；移除/关闭后门；CORS 收紧；协议法务审阅 | 2–3 周 |
| **Phase 2** | 可收费 MVP | 用户注册/登录；预充值 + 按次扣费；接入一种支付 | 4–6 周 |
| **Phase 3** | 正式商用 | 监控、测试、限流、帮助中心；套餐与定价优化 | 持续迭代 |

### 7.2 立即行动项（本周可完成）

1. **前端接入后端 AI**：增加 `VITE_APP_USE_BACKEND_AI=true` 开关，为 true 时前端调用 `api.plc-sim.com/api/ai/generate`，不传 Key；为 false 时保留现有直连逻辑（便于开发调试）
2. **后端鉴权占位**：`/api/ai/generate` 增加可选的 `Authorization: Bearer <token>` 校验，为后续用户体系预留
3. **CORS 收紧**：将 `*` 改为 `https://www.plc-sim.com`

### 7.3 不建议立即落地的原因

若在**未完成 Phase 1** 的情况下对外公开商用：

- API Key 暴露导致用户 Key 被盗用、平台声誉受损
- 无用户体系无法做计费与审计
- 协议未定稿存在法律风险
- 无监控与测试，线上故障难以快速定位
