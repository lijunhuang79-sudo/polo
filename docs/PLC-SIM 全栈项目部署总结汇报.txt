## Slide 6｜Nginx 配置关键点——HTTP 重定向

- server 块 1：统一入口 & 强制 HTTPS  
  - 监听 80 端口：  
    - `listen 80 default_server;`  
  - 重定向规则：  
    - 所有 `http://plc-sim.com` / `http://www.plc-sim.com`  
      → `https://www.plc-sim.com`  
- 目标：  
  - 提升安全性（所有访问走 HTTPS）  
  - 域名规范化，避免重复内容和乱跳转

---

## Slide 7｜Nginx 配置关键点——前端 SPA

- server 块 2：`www.plc-sim.com`  
- 主要配置要点：  
  - 监听：`listen 443 ssl http2;`  
  - 站点根目录：  
    - `root /var/www/plc-sim/frontend;`  
    - `index index.html;`  
  - SPA 路由支持（解决 500 / 重定向死循环问题）：  
    ```nginx
    location / {
        try_files $uri $uri/ /index.html;
    }
    ```  
  - 静态资源缓存优化：  
    ```nginx
    location ~* \.(js|css|png|jpe?g|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    ```  

> 重点说明：*删除了多余的 `/index.html` 重写逻辑，消除了 500 错误和重定向死循环*。

---

## Slide 8｜Nginx 配置关键点——API 反向代理

- server 块 3：`api.plc-sim.com`  
- 核心配置：  
  - 监听 HTTPS：`listen 443 ssl http2;`  
  - 反向代理到本地 Node.js 服务：  
    ```nginx
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;

        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    ```  
- 功能说明：  
  - 实现前端以 `https://api.plc-sim.com` 调用后端  
  - 保留真实客户端 IP 与协议信息（日志 & 鉴权更准确）

## Slide 9｜SSL 证书与 HTTPS 配置

- 证书来源：Let’s Encrypt  
- 统一证书路径：  
  - `ssl_certificate     /etc/letsencrypt/live/plc-sim.com/fullchain.pem;`  
  - `ssl_certificate_key /etc/letsencrypt/live/plc-sim.com/privkey.pem;`  
- 避免问题：  
  - 删除/避免使用多套证书路径（之前的冲突源）  
  - 所有 HTTPS server 块引用同一证书目录  
- 额外安全配置：  
  - `include /etc/letsencrypt/options-ssl-nginx.conf;`  
  - `ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;`

## Slide 11｜访问控制：Basic Auth 密码保护

- 目标：当前阶段仅自用，不对公众开放  
- 工具：Nginx Basic Auth  
- 步骤：  
  1. 创建密码文件：  
     ```bash
     sudo apt-get install apache2-utils -y
     sudo htpasswd -c /etc/nginx/.plc-sim-auth admin
     ```  
  2. 在前端 & API 的 server 块中添加：  
     ```nginx
     auth_basic "Restricted";
     auth_basic_user_file /etc/nginx/.plc-sim-auth;
     ```  
- 效果：  
  - 访问 `https://www.plc-sim.com` 和 `https://api.plc-sim.com` 时，浏览器弹出用户名/密码框  
  - 只有知道账号和密码的用户才能访问前端和接口

## Slide 12｜问题排查与关键修复记录

- 前端 500 错误 & 重定向循环  
  - 原因：Nginx 配置中存在 `/index.html` 重写死循环  
  - 解决：简化为单一规则：  
    - `try_files $uri $uri/ /index.html;`  
- SSL 证书异常 / 配置混乱  
  - 原因：多个 server 块引用不同证书路径，或重复申请证书  
  - 解决：统一使用 `/etc/letsencrypt/live/plc-sim.com/`  
- 访问控制未生效  
  - 现象：Basic Auth 未弹出密码框  
  - 排查：  
    - 核对实际生效的 Nginx 配置（`nginx -T`）  
    - 确认 `auth_basic` 所在 server 块与访问域名一致  
    - 检测响应头 `WWW-Authenticate` 返回情况  
  - 结果：已修复，密码框正常弹出
