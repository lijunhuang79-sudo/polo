# 前端部署清单 — 按顺序做完即完成前端所有部署

目标：将本仓库前端构建产物部署到 **https://www.plc-sim.com**（静态目录 `/var/www/plc-sim/polo/frontend`）。

---

## 一、在仓库/本机（只需做一次）

### 1. 确认生产环境变量

- 仓库中已有 **`env.production.example`**，内容包含：
  - `VITE_APP_API_BASE=https://api.plc-sim.com`
  - `VITE_APP_SKIP_LOGIN=false`
- 部署时若服务器上不存在 `.env.production`，会自动从该示例复制，无需提交敏感信息。

### 2. 确认构建与部署流程已就绪

- **GitHub Actions**：`.github/workflows/deploy.yml` 已在 push 到 `main` 时执行：
  - 在服务器上拉取代码 → `npm ci` → `npm run build` → 将 `dist/` 同步到 `/var/www/plc-sim/polo/frontend`
- **服务器本机脚本**：`deploy/scripts/deploy-polo.sh` 可单独在服务器上执行，完成同样步骤。

---

## 二、在 GitHub 上（只需做一次）

### 3. 添加部署用 SSH 私钥

1. 打开：**https://github.com/lijunhuang79-sudo/polo/settings/secrets/actions**
2. **New repository secret**
   - Name: **`DEPLOY_SSH_KEY`**
   - Value: 服务器 **root** 用户的 SSH 私钥全文（用于 213.111.157.18）

若服务器上还没有专用密钥，在服务器执行：

```bash
sudo -u root ssh-keygen -t ed25519 -C "polo-deploy" -f /root/.ssh/deploy_polo -N ""
cat /root/.ssh/deploy_polo.pub >> /root/.ssh/authorized_keys
sudo cat /root/.ssh/deploy_polo   # 复制整段到 GitHub DEPLOY_SSH_KEY
```

---

## 三、在服务器上（只需做一次）

### 4. 安装 Node.js 18+ 与依赖

```bash
# 示例：Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v   # 应 >= 18
```

### 5. Nginx 使用前端目录

任选一种方式，使 **www.plc-sim.com** 的 root 指向 `/var/www/plc-sim/polo/frontend`：

**方式 A**：使用仓库中的 Nginx 配置（推荐）

```bash
sudo cp /var/www/plc-sim/polo/deploy/nginx/plc-sim-polo.conf /etc/nginx/sites-available/plc-sim.conf
sudo ln -sf /etc/nginx/sites-available/plc-sim.conf /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

**方式 B**：只改现有配置里的 root

在现有 `plc-sim.conf` 中，把 **www** 站点的 `root` 改为：

```nginx
root /var/www/plc-sim/polo/frontend;
```

然后：

```bash
sudo nginx -t && sudo systemctl reload nginx
```

### 6. 首次部署：克隆仓库并执行一次部署脚本（若尚未克隆）

若服务器上还没有 `/var/www/plc-sim/polo`：

```bash
sudo mkdir -p /var/www/plc-sim
sudo chown $USER:$USER /var/www/plc-sim
git clone https://github.com/lijunhuang79-sudo/polo.git /var/www/plc-sim/polo
cd /var/www/plc-sim/polo
chmod +x deploy/scripts/deploy-polo.sh
./deploy/scripts/deploy-polo.sh
```

之后每次 push 到 `main` 会由 GitHub Actions 自动完成拉取、构建、同步到 `/var/www/plc-sim/polo/frontend`。

---

## 四、日常更新（仅前端时）

- **自动**：直接 **push 到 main**，Actions 会自动构建并把 `dist/` 同步到 `/var/www/plc-sim/polo/frontend`。
- **手动**：SSH 到服务器后执行：
  ```bash
  /var/www/plc-sim/polo/deploy/scripts/deploy-polo.sh
  ```

## 五、验证前端已部署成功

- [ ] 打开 **https://www.plc-sim.com**，输入 Basic Auth 后能正常打开页面、无白屏。
- [ ] 服务器上存在且为新文件：`ls -la /var/www/plc-sim/polo/frontend/index.html`、`ls /var/www/plc-sim/polo/frontend/assets/`。
- [ ] **支付图**：`public/alipay-payment.png` 会随构建复制到 `dist/`，部署后应能直接访问：**https://www.plc-sim.com/alipay-payment.png**。若该链接 404，说明部署时未包含此文件（确认 `public/alipay-payment.png` 已提交并推送到部署所用分支）。
- [ ] GitHub 仓库 **Actions** 页最近一次部署工作流为绿色成功。
