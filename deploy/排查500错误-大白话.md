# 输入密码后出现 500 错误 — 排查步骤（大白话）

你遇到的情况：打开 https://www.plc-sim.com → 弹出密码框 → 输入密码后出现 **500 Internal Server Error**。

---

## 原因说明（简要）

- 你用的是 **HTTPS**，Nginx 实际用的是 **SSL 那套配置**（如 `plc-sim.ssl.conf`）。
- 若 SSL 配置里前端的 **root** 还是旧路径 `/var/www/plc-sim/frontend`，而你现在前端文件在 **`/var/www/plc-sim/polo/frontend`**，就会导致：
  - 目录不存在或为空 → Nginx 找不到 index.html → **500**
  - 或目录权限不对 → Nginx 读不了 → **500**

---

## 一、先看 Nginx 报错（最准）

SSH 登录服务器后执行：

```bash
tail -30 /var/log/nginx/plc-sim-frontend.error.log
```

或：

```bash
tail -30 /var/log/nginx/error.log
```

看最后几行有没有类似：

- `No such file or directory` → 目录或文件不存在
- `Permission denied` → 权限不对
- `open() "/var/www/plc-sim/frontend/..." failed` → 说明用的还是旧路径

---

## 二、确认前端目录和内容

在服务器上执行：

```bash
ls -la /var/www/plc-sim/polo/frontend/
```

- 若提示 **No such file or directory**：说明部署脚本没把前端发布过去，需要重新跑一次 `./deploy/scripts/deploy-polo.sh`。
- 若有目录但**没有 index.html**：同样是部署未完成，再跑一次部署脚本。
- 若有 **index.html** 和 **assets**：说明文件在，问题多半是 Nginx 的 root 或权限。

再确认 Nginx 能读（一般用 www-data 用户）：

```bash
sudo -u www-data cat /var/www/plc-sim/polo/frontend/index.html | head -5
```

若这里报 Permission denied，需要改权限：

```bash
sudo chown -R www-data:www-data /var/www/plc-sim/polo/frontend
sudo chmod -R 755 /var/www/plc-sim/polo/frontend
```

---

## 三、改 HTTPS 配置里的 root（最常见）

你访问的是 **https**，所以必须改 **SSL 配置**里 www 的 root。

在服务器上执行：

```bash
grep -n "root " /etc/nginx/sites-enabled/*
```

看 **www.plc-sim.com** 对应的那个 server 里，**root** 后面跟的是不是 `/var/www/plc-sim/polo/frontend`。

若不是，编辑实际生效的配置（可能是 `plc-sim.conf` 或 `plc-sim.ssl.conf`），找到 **www.plc-sim.com** 的 server，把：

```nginx
root /var/www/plc-sim/frontend;
```

改成：

```nginx
root /var/www/plc-sim/polo/frontend;
```

保存后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

---

## 四、若你用的是仓库里的配置文件

- **HTTP**：用 `deploy/nginx/plc-sim-polo.conf`，里面已经是 `root /var/www/plc-sim/polo/frontend`。
- **HTTPS**：用 `deploy/nginx/plc-sim.ssl.conf`，仓库里已改为 `root /var/www/plc-sim/polo/frontend`。  
  在服务器上把最新版 `plc-sim.ssl.conf` 拷到 Nginx 的 sites-available，启用后 `nginx -t` 和 `reload`。

---

## 五、小结

| 现象 | 处理 |
|------|------|
| 日志里 path 是 `/var/www/plc-sim/frontend` | 把 HTTPS 配置里 www 的 root 改成 `/var/www/plc-sim/polo/frontend`，然后 `nginx -t`、`reload` |
| `polo/frontend` 目录不存在或没有 index.html | 在服务器上再执行一次 `./deploy/scripts/deploy-polo.sh` |
| Permission denied | `chown -R www-data:www-data /var/www/plc-sim/polo/frontend` 和 `chmod -R 755` |

按上面做完，再访问 https://www.plc-sim.com、输入密码，500 一般会消失。若仍有报错，把 `tail -30` 那段错误日志贴出来即可继续排查。
